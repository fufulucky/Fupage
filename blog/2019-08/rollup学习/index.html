Rollup.js学习（一）

一、什么是Rollup.js

Rollup.js是JavaScript的模块打包器，像是vue和react等框架都是通过rollup.js进行打包的。

二、第一个Rollup.js应用

1.安装rollup.js

全局安装：npm install rollup –global

2.创建一个简单的项目

首先来创建目录：mkdir src //在当前目录下创建src文件夹，接下来在src文件夹内创建一个main.js的文件，将下面的代码放入src/main.js中：

            import test from ‘./test.js’
            export default function() {
                console.log(test)
            }

然后在test.js中，对外暴露一个变量test

            const test = 1
            export default test

接下来就可以进行一个简单的打包：

输入指令：

rollup src/main.js –f cjs //（这里-f是必须带的，是—format的缩写，cjs是CommonJs标准，amd表示采用AMD标准，esm（或者es）为ES模块标准）

可以看到，cmd直接将结果打印在了命令提示符里。我们也可以通过指令来生成一个文件：

rollup src/main.js –f cjs –o dist/bundle.js //-o是—output.file的全称

打开dist目录下，看到bundle.js文件，

            ‘use strict’
            const test = 1
            function main() {
                console.log(test)
            }
            module.exports = main

可以看到rollup将两个文件进行了融合。

Rollup.js学习（二）

不用命令行，使用配置文件的方法

当要添加更多的选项时，这种方法就显得有些麻烦。为此，可以通过用JavaScript的语法写一个rollup.config.js的文件，增加如下代码：

            export default {
                input: ‘src/main.js’,
                output:{
                    file: ‘./dist/index-cjs.js’,
                    format: ‘cjs’
                }
            }

接下来，在cmd中使用rollup –config或者rollup –c来打包

打包出的文件与命令行的一样。

当要输出多个文件，或者使用rollup插件的时候，必须用配置文件的方法

Rollup.js学习（三）

打包一个vue项目来练手

首先新建一个文件夹，就命名为search吧，在search里建三个文件夹，分别为build，src和dist。其中build用来放置rollup.config.js文件，dist用来放打包后的文件。

在/src文件夹中，新建一个.vue文件和一个index.js文件

在.vue文件中，随便写点什么，然后给他一个name：test

在index.js中，作为项目入口，将vue文件导入

            import test from ‘./test.vue’
            const myCom = {
                install: function(Vue){
                    Vue.component(‘test’,test)
                }
            }
            export default myCom

接下来，在build中新建一个rollup.config.js文件，和前面一样，在这个文件中写清楚项目的入口和出口。另外，需要引入一些插件

            import vue from ‘rollup-plugin-vue’
            import commonjs from ‘rollup-plugin-commonjs’
            import buble from ‘rollup-plugin-buble’
            import resolve from ‘rollup-plugin-node-resolve’
            export default {
                input: ‘src/index.js’,
                output: {
                    format: ‘cjs’,
                    file: ‘./dist/myComponent.js’
                },
                plugins: [
                    commonjs(),
                    vue(),
                    buble(),
                    resolve()
                ]
            }

其中commonjs，buble，resolve都是项目所需的插件。

为什么需要这些插件呢？

对于commonjs来说，因为rollup.js默认不支持commonjs模块

Buble可以支持ES6的新特性

Resolve可以下载远程npm库

接下来在命令提示符里，输入 npm init

在生成的package.json中，增加一条：

            “build”: “rollup –config build/rollup.config.js”,

这样就可以直接用 npm run build来执行打包语句。

为了能在项目中直接引用，还需要更改一条

              “main”: “dist/searchCom.js”,

Mian里是打包项目的入口。

在命令提示符里打包，输入npm run build，成功生成打包文件myCompinent.js

第二部分—将打包文件发布到npm中，在别的项目里只需要npm install，再在main.js中import和Vue.use()就可以方便的使用组件了

发布到npm中：

1. 登录npm 指令： npm login

2. 检查一下package.json中的name，在npm网站上有没有重复的，如果重复就改掉。

3. 发布 指令：npm publish

此时已经将我们的vue组件发布到npm中了。

在本地文件中，如果要使用发布的这个包，首先npm install xxxxx 然后在main.js中引用 

            import  myCom  from ‘../node_modules/rollup2.1/dist/myComponent’
            Vue.use(myCom)

接下来就可以在组建中使用了
